ВЫПОЛНЕНИЕ СКРИПТА:

1) bash scriptname
2) source scriptname
3) ./scriptname
# Третий способ работает, если в начале скрипта указать то, как выполнять этот скрипт:
# например, указать строку: #!/bin/sh, а затем сделать файл испольняемым: chmod a+x scriptname


ПЕРЕМЕННЫЕ:
к переменным можно обращаться только по ссылкам:
1) $variable		- стандартный способ
2) ${variable}		- более строгий способ

присваивание значения перменной:
variable=123 		# БЕЗ ПРОБЕЛОВ
echo $variable		# вывести значение
# если просто вызвать $variable, то shell вернёт значение переменной, а затем 
# будет пытаться выполнить это значение как команду

ОБЩИЕ ПЕРЕМЕННЫЕ:
$? - содержит код возврата (или ошибки)
$* - содержит все позиционнные аргументы скрипта
$@ - содержит все позиционнные аргументы скрипта
$# - содержит количество параметров
$_ - последний аргумент предыдущей команды
$$ - содержит PID процесса/скрипта (или PID терминала, если вызвать терминал)

ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ:
export VALUE=25			- создание переменной окружения
printenv			- посмотреть перменные окружения


ПЕРЕЧИСЛЕНИЯ (типа списки):
echo {5..12} 		# 5,6,7,8,9,10,11,12	(от одного до пяти)
echo {1..5..2}		# 1,3,5 		(от одного до пяти с интервалом 2)

можно использовать так
mkdir /path/{foo,bar,var,jar} # по пути path создастся 4 папки


УСЛОВИЯ (ПРОВЕРКА УСЛОВИЙ)

[[ $a = 1 ]]	- сравнивает переменую a как ЧИСЛО с числом
[[ "$a" == 1]]	- вроде как делает то же самое 
(( $a == 1 ))	- сравнивает переменую a как ЧИСЛО с числом

в ((  )) нельзя подставлять -eq, -ne и прочие такие штуки
