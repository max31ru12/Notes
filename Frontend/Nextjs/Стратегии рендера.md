

## SSG - Static Site Generation

1. next build
2. next выполняет страницу
3. создается html-файл
4. кладется в .next
5. Сервер отдается готовый html

Если страница не использует динамические данные, то Next делает ее SSG

##### Пример SSG-страницы

```tsx
export default function Page() {  
return <div>Hello</div>  
}
```



## SSR - Server Side Rendering

HTML генерируется на сервере при каждом запросе

1. User request
2. Сервер запускает React Render
3. Генерируется HTML
4. Отдается клиенту

##### Пример SSR-страницы

```tsx
export const dynamic = "force-dynamic"

export default async function Page() {
  const data = await fetch("https://api.site.com/posts")
  return <div>{data}</div>
}
```

Во время `next build` html не генерируется


### Сделать SSR принудительно

1. Убрать `use client`
2. Добавить `export const dynamic = "force-dynamic";`

## ISR - Incremental Static Regeneration

Страница генерируется на build, но **может обновляться позже**

1. build -> html
2. через N секунд
3. перегенерация


##### Пример

```tsx
export const revalidate = 60
```

HTML кэшируется 60 секунд


## CSR - Client Side Rendering


##### Пример

```tsx
"use client"

export default function Page() {
  const [data, setData] = useState()
}
```

### Принудительный CSR

```tsx
export const dynamic = "force-dynamic"  
  
"use client"  
  
export default function Page() {  
	return <div>Legal documents</div>  
}
```

1. `export const dynamic = "force-dynamic"` принудительно делает SSR вместо SSG
2. `use client` сообщает о том, что это клиентский компонент


## Streaming (React Server Components)

Сервер может отправлять HTML частями

1. Request
2. render header
3. send HTML
4. render slow component
5. send remaining HTML
