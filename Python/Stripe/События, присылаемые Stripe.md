
| Событие                                    | Что значит                                                                                                                                                                                                                                                 | Когда обрабатывать                                                                                                                                                                                                                    |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `checkout.session.completed`               | Пользователь успешно завершил Checkout Session. Для обычных instant-методов это основной сигнал, что checkout закончен. ([Документация Stripe](https://docs.stripe.com/api/events/types?utm_source=chatgpt.com "Types of events \| Stripe API Reference")) | Обрабатывать для логики уровня checkout: связать сессию с заказом, сохранить `session.id`, `customer`, `payment_intent`, обновить внутренний статус. Для instant payments можно использовать как триггер успешного завершения заказа. |
| `checkout.session.async_payment_succeeded` | Платеж по Checkout Session с **delayed payment method** окончательно подтвердился позже. ([Документация Stripe](https://docs.stripe.com/api/events/types?utm_source=chatgpt.com "Types of events \| Stripe API Reference"))                                | Обрабатывать как финальный успех для асинхронных методов оплаты. Здесь безопасно выдавать доступ, активировать подписку, подтверждать заказ.                                                                                          |
| `checkout.session.async_payment_failed`    | Платеж по Checkout Session с delayed payment method позже окончательно провалился. ([Документация Stripe](https://docs.stripe.com/api/events/types?utm_source=chatgpt.com "Types of events \| Stripe API Reference"))                                      | Обрабатывать как финальный fail для асинхронных методов: помечать оплату failed, не выдавать товар/доступ, просить другой способ оплаты.                                                                                              |
| `payment_intent.succeeded`                 | Сам `PaymentIntent` успешно оплачен. Это событие именно про платеж. ([Документация Stripe](https://docs.stripe.com/payments/payment-intents/verifying-status?utm_source=chatgpt.com "Payment status updates"))                                             | Обрабатывать для платежной логики: финально помечать payment как succeeded, сохранять данные провайдера, сумму, currency, charges. Часто это лучший источник истины именно для статуса платежа.                                       |
| `payment_intent.payment_failed`            | Попытка оплаты по `PaymentIntent` провалилась. ([Документация Stripe](https://docs.stripe.com/payments/payment-intents/verifying-status?utm_source=chatgpt.com "Payment status updates"))                                                                  | Обрабатывать для платежной логики: ставить payment в failed, сохранять код/причину ошибки, инициировать retry или просить другой payment method.                                                                                      |
