Параметры пути имеют вид: .../path/{parameter}
---- для их валидации используется Path(..., )
---- ... - означает, что параметр обязательный (Path в любом случае обязательный, можно не указывать)
---- есть всякие штуки типа ge, le и тд (валидацию можно производить)



Параметры запроса имеют вид .../path?param=123
---- для их валидации используется Query()
---- тоже позволяет производить валидацию


body:
---- для их валидации можно использовать Body -- value: Annotated[int, Body()]
---- запрос: "value": 5
	

---- если несколько параметров указано: item: Item, model: Model, то в запросе надо указывать так:
---- {
      "item": {...},
      "model": {...},
---- }

---- в Body можно прописать embed=True, чтобы ждать не {...}, а "param_name" : {...}


Вместо этого всего использовать Field() КОГДА НУЖНА БОЛЕЕ ДЕТАЛЬНАЯ НАСТРОЙКА
